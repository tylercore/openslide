# AI Stock 產品業務推廣規劃書

> 版本：v1.0｜日期：2026-07-14  
> 產品定位：標準化、邀請制的 AI 投資研究成果平台

## 一、規劃結論

AI Stock 第一階段不再以「股票交易機器人」作為主產品，而是服務機構與專業投資經理人的研究成果平台。

產品先將通過資料驗證與研究審核的「標的特性」做成固定模板，依既定台股 universe 每日批次執行，將全市場結果以一致介面呈現。客戶使用邀請制帳號查看最新與歷史研究成果，再依其內部流程使用研究結果。

首版目標是讓業務團隊可以穩定展示、讓客戶可以實際使用與回訪；不以完整商業服務、券商串接或自動交易作為首版上線條件。

## 二、產品定位與邊界

### 目標客群

- 機構投資研究團隊
- 專業投資經理人
- 需要定期建立候選標的池的投資決策團隊

### 核心問題

客戶需要從全市場快速找出具備特定特性的候選標的，但現有流程可能依賴人工篩選、零散腳本或難以重跑的研究紀錄，導致研究耗時、結果不一致，也不容易向團隊說明判斷依據。

### 核心價值主張

> 用已驗證、可追溯、可重跑的標的特性，縮短投資團隊建立候選標的池的時間。

### 首版包含

1. 固定版本的已驗證標的特性模板，初期規模為 3–5 個。
2. 既定台股 universe 的每日收盤後批次研究。
3. 研究結果工作台：排名、分數、等級、樣本數、證據、限制與歷史結果。
4. 資料來源、資料版本、模板版本、研究規格與執行紀錄的追溯。
5. 邀請制帳號與基本的機構資料隔離、唯讀查看及使用追蹤。
6. 業務可重複使用的標準 Demo 流程與展示資料。

### 首版不包含

- 券商串接、自動下單與交易執行
- 通知與盤中即時監控
- 客戶自行建立任意研究特性
- 客戶自行即時執行研究
- 研究結果檔案交付與外部資料下載
- 直接提供投資建議或報酬保證
- 首版即承諾完整 API 整合、SLA 或客製化研究

通知、交易工作台與券商整合可保留為後續加分能力，但不應阻塞 AI Stock 的首版上線。

## 三、產品使用流程

```text
已驗證特性模板
        ↓
每日收盤後批次研究
        ↓
結果驗證與版本紀錄
        ↓
研究結果工作台
        ↓
查看排名／分數／等級／證據
        ↓
平台內查看與判讀
        ↓
客戶內部研究或交易流程
```

標準 Demo 也沿用同一流程：選擇研究模板 → 查看全市場結果 → 解釋排名與可信度 → 篩選候選標的 → 說明如何銜接客戶既有研究流程。

## 四、研究特性治理

每個特性進入正式特性庫前，必須完成以下審核：

- 使用明確資料來源、資料期間與資料版本。
- 具備足夠樣本數，並保留研究規格與執行紀錄。
- 將訓練／研究期間與驗證期間分離，降低過度擬合風險。
- 檢查跨期間或不同市場狀態下的穩定性。
- 提供分數、等級、樣本數、診斷資料與限制說明。
- 由研究／產品負責人完成核准並建立特性版本。

對外溝通時，應說明特性是研究篩選工具，不宣稱必然報酬，也不將歷史結果直接等同未來績效。

## 五、V1 開發路線

以下以 12 週、可平行推進為規劃基準；實際日期依人力與資料管線現況調整。

### Phase 0｜重新定義與基線確認（第 1–2 週）

目標：將模糊產品收斂成可展示的第一版。

- 凍結產品邊界：研究成果平台，不含交易執行。
- 確認台股 universe、資料更新時間與資料品質檢查。
- 選定首批 3–5 個特性模板及其研究負責人。
- 定義研究結果資料模型：標的、分數、等級、樣本數、證據、限制、版本與執行狀態。
- 定義標準 Demo 案例、展示資料與業務話術。

完成條件：產品規格、結果欄位、特性審核表與 Demo 腳本獲得產品／研究／業務共同確認。

### Phase 1｜研究結果管線（第 3–5 週）

目標：能穩定產出可追溯的每日研究結果。

- 建立固定特性模板與版本管理。
- 建立每日批次執行、失敗狀態與重跑機制。
- 寫入資料來源、資料版本、模板版本、研究規格與執行時間。
- 完成結果品質檢查與審核狀態。
完成條件：既定 universe 可完成連續批次執行，結果可重跑、可比較，且失敗不會產生未標示的半成品結果。

### Phase 2｜標準平台與帳號（第 6–9 週）

目標：讓客戶能以一致介面查看成果。

- 建立邀請制帳號與機構資料隔離。
- 完成最新結果、歷史結果與模板版本查看。
- 呈現排名、分數、等級、樣本數、證據與限制。
- 提供候選標的篩選與研究結果瀏覽。
- 補齊空結果、批次執行中、失敗、資料不足等狀態。
- 建立產品使用紀錄，支援後續回訪分析。

完成條件：受邀使用者不依賴產品人員逐頁解釋，即可完成查看研究結果、理解證據並篩選候選標的。

### Phase 3｜業務展示上線（第 10–12 週）

目標：讓業務可對外進行穩定、可重複的標準 Demo。

- 完成 Demo tenant、展示帳號與固定展示資料。
- 完成 15–20 分鐘標準 Demo 腳本。
- 準備產品一頁簡介、研究特性說明、結果解讀指南與常見問答。
- 進行內部業務演練與至少一次端到端故障演練。
- 開放邀請制客戶查看成果，開始追蹤實際使用與回訪。

完成條件：業務可獨立完成 Demo；客戶可取得帳號、查看每日成果並回看歷史版本。

## 六、業務推廣方案

### 推廣主訊息

主標題：

> 讓投資團隊更快建立可解釋、可追溯的候選標的池。

三個支撐訊息：

1. 不是黑盒推薦：每項結果都有特性、分數、樣本數、資料與限制說明。
2. 不是一次性報告：研究以固定模板每日批次更新，保留歷史版本。
3. 不需改造交易流程：先用平台取得研究成果，再由客戶帶回既有研究或交易平台。

### 標準 Demo 腳本

1. 先問客戶目前如何從全市場建立候選標的池，以及研究結果如何交接給投資／交易團隊。
2. 展示一個已驗證特性，說明資料期間、研究規格與適用限制。
3. 展示每日結果的排名、分數、等級與樣本數。
4. 點開單一標的，說明它為何被分級，以及哪些證據仍需人工判斷。
5. 回看歷史批次，展示模板版本與結果可追溯性。
6. 說明客戶如何將平台研究結果帶回既有內部流程。
7. 以「邀請制平台使用」作為下一步，而非立即承諾客製化。

### 目標客戶判斷條件

優先接觸符合以下條件的團隊：

- 有固定的台股研究或候選標的建立流程。
- 每週或每日需要重新篩選市場標的。
- 已經使用表格、腳本或人工流程，但結果難以追蹤或交接。
- 願意讓多位研究／投資角色共同查看同一份研究成果。
- 能在 Demo 後安排實際帳號使用與回訪。

### 業務轉換路徑

```text
標準化業務 Demo
        ↓
邀請制帳號
        ↓
4 週實際使用觀察
        ↓
每週回訪與研究流程盤點
        ↓
確認成果是否進入客戶內部流程
        ↓
正式方案／高階研究服務／後續客製化
```

第一階段不以簽約或營收作為唯一成功條件，而以客戶是否持續使用、回看結果及將研究判斷帶回內部流程作為產品市場驗證依據。

## 七、成效指標

### 產品穩定性

- 每日批次成功率與完成時間。
- 失敗批次的可辨識率與重跑成功率。
- 結果具備完整資料、模板、研究規格與執行版本。
- 平台顯示結果與研究批次結果一致。

### 客戶使用

以每個受邀機構 4 週為一個觀察週期，至少追蹤：

- 每週是否回訪平台。
- 是否查看最新研究結果。
- 是否查看歷史結果或單一標的證據。
- 是否查看單一標的的證據與限制說明。
- 客戶是否能指出研究結果如何進入內部研究／投資流程。

建議初期用「每週活躍機構數、結果查看率、證據查看率、4 週留存率」作為主儀表板，不以登入次數作為唯一指標。

### 業務推廣

- 完成標準 Demo 的業務人員數。
- 每月完成的有效機構 Demo 數。
- Demo 後取得邀請制帳號的比例。
- 受邀客戶的 4 週回訪率。
- 客戶提出的需求中，可由標準產品滿足的比例。

## 八、風險與處理方式

| 風險 | 影響 | 處理方式 |
| --- | --- | --- |
| 券商串接尚未談定 | 交易產品進度受阻 | 從首版範圍移除；先以平台瀏覽研究成果 |
| 特性回測只在單一期間有效 | 客戶信任下降 | 設定跨期間驗證與研究限制說明，通過審核才進特性庫 |
| AI 結果被誤解為投資建議 | 合規與商譽風險 | 對外明確定位為研究篩選工具，不保證報酬 |
| 每日批次失敗或延遲 | 客戶無法信任每日結果 | 提供批次狀態、失敗提示、重跑機制與最後成功更新時間 |
| 客製化需求過早湧入 | 產品路線分散 | 先記錄需求，只有能服務多數目標客戶的能力才進入標準產品 |
| 客戶只看 Demo、不持續使用 | 無法驗證產品價值 | 以邀請制帳號與 4 週使用觀察承接 Demo，安排固定回訪 |

## 九、後續版本方向

只有在 V1 的客戶使用證據成立後，才依優先級評估：

1. 研究特性持續新增與特性庫分級。
2. 客戶可調整研究期間、條件或建立有限度的自訂研究規格。
3. 提供 API 給內部交易平台或其他研究系統。
4. 加入候選標的通知與監控。
5. 與內部交易平台整合，縮短研究成果到執行的交接距離。
6. 評估券商串接與交易能力是否值得納入產品線。

## 十、目前仍需在執行前確認的事項

這些事項不阻塞產品定位，但需要在 Phase 0 完成：

- 首批 3–5 個特性的正式名稱、研究假設與負責人。
- 台股 universe 的正式範圍與每日資料可用時間。
- 研究結果的權限層級與機構資料隔離規則。
- 4 週使用觀察的邀請客戶名單與業務負責人。
- Demo 使用的固定案例、免責聲明與結果解讀口徑。
- 正式商業方案、定價與 SLA；先不作為 V1 展示上線的阻塞條件。

## 一句話版本

AI Stock 先不做交易，而是用已驗證的標的特性，每日產出可追溯的全市場研究成果，透過邀請制平台讓機構投資團隊查看與解釋候選標的，先以標準 Demo 與實際使用驗證市場，再逐步擴充特性、API、通知與交易整合。
